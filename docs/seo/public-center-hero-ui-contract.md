# Public center hero UI contract

The public center profile hero is a presentation layer only.

It may show:

- the public center profile badge
- the center name as the page `h1`
- a short fact-based profile summary
- reviewed public contact actions when available
- neutral metadata chips such as center type, public location summary, public profile state, and contact review state

It must not show:

- ratings or review counts
- booking claims
- opening status claims
- insurance claims
- official regulator approval claims
- emergency availability claims
- unreviewed contact details

The hero must keep the existing metadata, robots, sitemap, and profile index gates unchanged.

The long provider or editorial description belongs in the profile body, not in the first-fold hero copy.
