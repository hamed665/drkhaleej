# Public Provider Canonical Route Contract

Status: canonical route resolver contract.
Scope: documentation and validation only.
Build mode: resolver-first, route-last, fail-closed.

## Purpose

Public provider URLs must be generated by one resolver before internal links, sitemap promotion, hreflang, payload projection, shared cards, or imported hospital release can use them.

Cards, search results, sitemaps, schema builders, and page payload builders must not construct provider URLs directly.

## Route families

The resolver must support these route families as policy values:

```text
doctor
center
hospital
clinic
dental_clinic
dentist
pharmacy
lab
imaging_center
beauty_clinic
charity_center
pet_clinic
pet_shop
```

The existence of a route family in the resolver does not mean the public route is enabled.

## Current enabled detail routes

The current enabled detail routes are:

```text
doctor → /:locale/:country/doctor/:slug
center → /:locale/:country/center/:slug
```

All other families must be fail-closed until their dedicated integration PR enables them.

## Resolver result

The resolver must return:

```ts
{
  canonicalPath: string | null;
  routeFamily: PublicProviderRouteFamily;
  publicRouteEnabled: boolean;
  reason: PublicProviderRouteReason;
}
```

## Required reasons

```text
enabled
unsupported_family
missing_slug
route_disabled
invalid_locale
invalid_country
```

## Fail-closed rules

The resolver must return `canonicalPath: null` and `publicRouteEnabled: false` when:

```text
family is unsupported
slug is empty or invalid
locale is unsupported
country is unsupported
family route is disabled
```

## Canonical path rules

Canonical paths must be locale/country-aware.

Allowed current outputs:

```text
/en/om/doctor/[slug]
/ar/om/doctor/[slug]
/en/om/center/[slug]
/ar/om/center/[slug]
```

The resolver must not output imported hospital detail paths until imported hospital controlled release gates pass.

## Public surface rules

These modules may consume resolver results:

```text
internal link engine
hreflang projection
sitemap eligibility gate
page payload projection
shared provider card view model
search result view model
schema visibility layer
```

These modules must not generate provider URLs independently.

## Imported hospital rule

Imported hospital routes remain blocked.

A hospital family result may exist as a typed route family, but `publicRouteEnabled` must remain false until:

```text
imported hospital release blockers pass
canonical resolver is wired
internal link coverage exists
hreflang projection is ready
sitemap eligibility gate passes
page payload projection is used
shared provider card view model is used
performance/client boundary guard passes
first indexable batch is complete
```

## Non-goals

This contract does not add or change:

- public routes
- sitemap output
- hreflang output
- internal link candidates
- payload projection
- card rendering
- imported hospital public release
- database schema
